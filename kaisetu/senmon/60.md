# 問1：正解2

## 1. 3つの重要キーワードを日常の言葉に訳そう

まずは、問題に出てくる専門用語を身近なイメージに置き換えてみましょう。

- **飽和水蒸気圧** → **「部屋の定員（満員状態のときに入れる水分の限界量）」**
    - ※温度が高くなるほど、空気という部屋が大きくなり、定員（入れる水分）が増えます。
- **水蒸気量（水蒸気圧）** → **「実際に部屋の中にいる水分の人数」**
- **相対湿度** → **「部屋の混雑度（定員に対して、何%の人がいるか）」**
- **露点（ろてん）** → **「部屋の人数はそのままで、部屋を冷やしていったときに、ちょうど満員（湿度100%）になって水滴がつき始める温度」**
- **乾球温度と湿球温度** → 乾球温度は「ふつうの気温」**、湿球温度は**「濡らした温度計の温度」です。
    - 空気が乾燥しているほど水分がよく蒸発して熱を奪うため、2つの温度の差（引き算）が大きくなります。

## 2. 「これだけ！」のシンプルルール

- **ルール①：** 水蒸気量（実際の人数）を知りたければ、「露点」のときの飽和水蒸気圧（定員）を見れば一発でわかる！
- **ルール②：** 相対湿度（混雑度）＝ **「実際の人数（水蒸気圧）÷ 定員（その気温の飽和水蒸気圧）× 100」**

## 3. シミュレーションで各地点のデータを整理しよう

実験のように、A、B、Cそれぞれの「実際の人数（水蒸気圧）」と「混雑度（相対湿度）」を計算してみましょう。

### 地点A：気温20℃、湿度70%

- **定員（20℃の飽和水蒸気圧）：** 表1より **23.4 hPa**
- **混雑度（相対湿度）：** **70%**
- **実際の人数（水蒸気圧）：** 23.4 hPa の 70% なので、
    
    $23.4 \times 0.7 = 16.38 \text{ hPa}$
    

### 地点B：気温25℃、露点15℃

- **実際の人数（水蒸気圧）：** 「露点が15℃」ということは、15℃のときの定員が、今の実際の人数と同じということです。表1で15℃のところを見ると **17.1 hPa** です。
- **定員（25℃の飽和水蒸気圧）：** 表1より **31.7 hPa**
- **混雑度（相対湿度）：** 実際の人数 ÷ 定員 なので、
    
    $17.1 \div 31.7 \times 100 \fallingdotseq 53.9\%$
    

### 地点C：乾球温度（気温）20℃、湿球温度15℃

- **温度の差：** 乾球20℃ − 湿球15℃ ＝ **5℃**
- **混雑度（相対湿度）：** 表2を使います。「乾球温度20」の行と、「温度差5」の列が交わるところを見ると **59%** です。
- **定員（20℃の飽和水蒸気圧）：** 表1より **23.4 hPa**
- **実際の人数（水蒸気圧）：** 23.4 hPa の 59% なので、
    
    $23.4 \times 0.59 = 13.806 \text{ hPa}$
    

### 三つの地点の結果まとめ（比較表）

| **地点** | **気温** | **実際の水蒸気量（人数）** | **相対湿度（混雑度）** |
| --- | --- | --- | --- |
| **A** | 20℃ | **16.38 hPa** | 70% |
| **B** | 25℃ | **17.1 hPa** (最多) | **53.9%** (最低) |
| **C** | 20℃ | **13.81 hPa** (最少) | 59% |

## 4. (a)〜(c)の正誤を判定！

作成したまとめ表を見ながら、問題文をチェックしましょう。

- **(a) 水蒸気量が最も少ないのは、地点Cである。**
    
    → 地点Cは 13.81 hPa で一番少ないので、文は **正（○）** です。
    
- **(b) 水蒸気量が最も多いのは、地点Aである。**
    
    → 最も多いのは地点B（17.1 hPa）なので、文は **誤（×）** です。
    
- **(c) 相対湿度が最も低いのは、地点Bである。**
    
    → 地点Bは 53.9% で一番低いので、文は **正（○）** です。
    

よって、組み合わせは **「正・誤・正」** となり、選択肢は **②** が正解になります！

## 1. 「正体」を日常の言葉に訳す（概念の翻訳）

まずは、気象レーダーの世界の専門用語を「日常のイメージ」に置き換えてみましょう。

- **気象レーダー** → **「電波を飛ばして、返ってきた跳ね返り（エコー）で雨や雪の強さを測るマシン」**
- **融解層（ゆうかいそう）** → 上空の雪が落ちてきて、**「ちょうど雨に融け変わる高さの境界線」**
- **ブライトバンド** → レーダー画面に現れる「やけにギラギラ光る強い電波の輪っか（明るい帯）」

## 2. 「ルール」をシンプルに提示する（変数の整理）

気象レーダーの電波には、「どんな粒にぶつかると、強く跳ね返るか」というシンプルな2つのルールがあります。

- **ルール①：大粒なほどよく反射する**
    - 小さな粒よりも、ドカンと大きな粒の方が電波をガッツリ跳ね返します。
- **ルール②：雪（固体）より水（液体）の方がよく反射する**
    - 電波は、乾いた雪の粒よりも、濡れた水の粒（液体）の方がはるかに反射しやすい性質を持っています。

## 3. 「シミュレーション」で解く（ステップ式解説）

この2つのルールを頭において、上空から落ちてくる雪の気持ちになって(a)〜(d)を順番にチェックしていきましょう！

### (a) 「みぞれ」の状態について

- **問題文：** 雨滴よりも粒が大きく、雪の表面が液体で覆われている「みぞれ」である。
- **解説：** 上空の雪（ひらひらして大きい）が融け始めると、芯は雪のまま、表面だけが水で濡れた大きな粒（みぞれ）になります。完全に融けて雨になると、水滴はギュッと小さく引き締まります。つまり、融けかけのみぞれは「雨よりも粒が大きい」というのは正しいです。
- **結論：** **正（○）**

### (b) レーダーの電波の反射しやすさについて

- **問題文：** 液体の状態であるよりは、固体である方が電波をよく反射する。
- **解説：** ルール②を思い出してください。電波は「固体（雪）よりも液体（水）の方がよく反射する」という性質があります。問題文は逆のことを言っています。
- **結論：** **誤（×）**

### (c) この環状の強いエコーの名前について

- **問題文：** 「エンゼルエコー」と呼ばれている。
- **解説：** このように雪から雨に変わる場所（みぞれの層）で、「水に濡れていて（液体）」「粒が大きい」**という電波が大好物の条件が揃うため、レーダーがそこだけ激しく反応してギラギラした輪っかを作ります。この現象を**「ブライトバンド（明るい帯）」と呼びます。
    - ※ちなみに「エンゼルエコー（天使の反射）」は、雨が降っていない晴れた日に、虫や空気の密度の差などにレーダーが反応してしまう、まったく別の現象です。
- **結論：** **誤（×）**

### (d) なぜ「輪っか」になるのかについて

- **問題文：** 融解層がほぼ一定の高度で水平方向に広がっていたことを示している。
- **解説：** レーダーは、傾き（仰角）をつけてぐるっと360度ビームを回して観測しています。空に向けて斜めに放たれた電波が、水平に広がっている「みぞれの層（融解層）」を突き抜けるとき、ドーナツ状にそこだけ強く反射することになります。きれいな輪っかになっているということは、その層が**平らにきれいに広がっている証拠**です。
- **結論：** **正（○）**

## 4. 必勝のコツ（記憶の定着）

判定を並べると、**(a) 正、(b) 誤、(c) 誤、(d) 正** となり、選択肢の **③** が見事に導き出せます。

# 問5：正解5

## 1. 「正体」を日常の言葉に訳そう

まずは、専門用語をイメージしやすい言葉に翻訳しましょう。

- **格子間隔（こうしかんかく）** → コンピュータの「画面の画素数（ピクセル）の粗さ」
- **非静力学モデル** → 積乱雲のように「空気が激しく上下にドカンと動く現象もガチで計算できる高度なプログラム」
- **診断的に計算する** → 過去のデータや他の数字から「これなら、きっとこうだろうと（時間変化を追わずに）その場で割り出すこと」
- **予報的に計算する** → 「今の状態がこうだから、1秒後はこう、2秒後はこう……」と「時間の流れに沿ってガチでシミュレーションすること」
- **時間ステップ** → パラパラ漫画の「次のページをめくる時間の間隔」

## 2. 「ルール」をシンプルに提示する

コンピュータで波や天気を計算するときには、絶対に破れない鉄のルールがあります。

- **ルール①：網目（格子間隔）の「5〜8倍以上」の大きさがないと、形がガタガタになってきれいに表現できない。**
- **ルール②：激しい上下運動（積乱雲など）を計算するときは、上下の動きも「予報的（時間変化を追う）」にガチ計算しなければならない。**
- **ルール③：パラパラ漫画をめくる間隔（時間ステップ）が長すぎると、絵がワープしすぎて計算が大爆発（破綻）する（CFL条件）。**

## 3. シミュレーションで(a)〜(c)を判定！

このルールに沿って、(a)〜(c)の下線部を実況中継しながらチェックします。

### (a) 表現できる現象の大きさについて

- **問題文：** 精度よく表現しうる現象は、水平スケールが格子間隔と同程度以上の現象である。
- **解説：** たとえば、デジタルカメラの画素（ピクセル）を思い浮かべてください。1ピクセルと同じ大きさのものを撮影しようとしても、ただの1点になってしまい、どんな形をしているか（精度よく）表現できませんよね。天気予報の網目でも同じで、格子間隔の「数倍（5〜8倍）以上」の大きさがないと、低気圧や高気圧の渦を正しく表現できません。「同程度（1倍）以上」では小さすぎて無理なのです。
- **結論：** **誤（×）**

# 問14：正解2

## 1. 「正体」を日常の言葉に訳す（概念の翻訳）

まずは、統計の専門用語を「日常のイメージ」に置き換えるところからスタートします。

- **平均誤差（ME）** → **「予報のクセ（高めに出しやすいか、低めに出しやすいか）」**
    - ※ 計算はシンプルに「予報 − 実況」の平均です。プラス（正）なら高めに見積もるクセ、マイナス（負）なら低めに見積もるクセがあると分かります。
- **2乗平均平方根誤差（RMSE）** → プラスマイナスを無視した「純粋な外しっぷりの平均（ズレの平均値）」
    - ※ 単純な引き算だとプラスとマイナスで打ち消し合ってしまうため、2乗して全員プラスにしてから平均をとる方法です。
- **真夏日** → 最高気温が **「30℃以上」** の日のこと。
- **見逃し率** → 実際は真夏日だったのに「真夏日じゃない（30℃未満）と予報して外してしまった割合」。

## 2. 「ルール」をシンプルに提示する（変数の整理）

この問題を解くための、計算のシンプルな下準備ルールはこれだけです。

- **ルール①：すべての日の「予報 − 実況」の引き算シートを作ろう！**
- **ルール②：真夏日（30℃以上）の欄だけをマーキングして、「見逃し（実況は30℃以上なのに、予報が20歳台）」を探そう！**

## 3. 「シミュレーション」で解く（ステップ式解説）

ルール①に従って、A地点・B地点それぞれの「予報 − 実況（予報誤差）」を計算してみましょう。

### ステップ1：日ごとのズレ（予報 − 実況）を計算

- **A地点のズレ**
    - 1日：$27 - 23 = +4$
    - 2日：$27 - 29 = -2$
    - 3日：$29 - 32 = -3$
    - 4日：$34 - 33 = +1$
    - 5日：$33 - 28 = +5$
    - 6日：$32 - 34 = -2$
    - 7日：$30 - 32 = -2$
    - **合計：** $+4 - 2 - 3 + 1 + 5 - 2 - 2 = +1$
- **B地点のズレ**
    - 1日：$30 - 26 = +4$
    - 2日：$28 - 30 = -2$
    - 3日：$31 - 33 = -2$
    - 4日：$34 - 32 = +2$
    - 5日：$30 - 30 = 0$
    - 6日：$29 - 31 = -2$
    - 7日：$30 - 28 = +2$
    - **合計：** $+4 - 2 - 2 + 2 + 0 - 2 + 2 = +2$

### ステップ2：(a)〜(c)を順番にジャッジ！

### (a) 平均誤差（ME）の偏りについて

- **判定：**
    - A地点のズレの合計は「+1」なので、平均してもプラス（正）です。
    - B地点のズレの合計は「+2」なので、平均してもプラス（正）です。
- **結論：** どちらの地点も「正の偏り」があるため、文は **正（○）** です。

### (b) 2乗平均平方根誤差（RMSE：純粋な外しっぷり）の比較

2乗を細かく計算しなくても、日ごとのズレの「絶対値（数字の大きさ）」を比べるだけで一目瞭然です。

- **A地点のズレの大きさ：** 4, 2, 3, 1, 5, 2, 2 → 「5」や「4」など、**大外ししている日があります。**
- **B地点のズレの大きさ：** 4, 2, 2, 2, 0, 2, 2 → 1日目以外は「2」か「0」に収まっており、**かなり安定しています。**
- **結論：** 明らかにA地点の方が誤差が大きく、B地点の方が小さいです。問題文は「B地点の方が大きい」となっているため、 **誤（×）** です。

### (c) 真夏日（実況30℃以上）の「見逃し率」の比較

ルール②の通り、実況が30℃以上の日のうち「予報が30℃未満だった日」をカウントします。全体の予報数はどちらも「7」です。

- **A地点：** 実況が30℃以上の日は【3日(32), 4日(33), 6日(34), 7日(32)】の5日間。
    - このうち予報が30℃未満（見逃し）だったのは**3日（予報29）の1回**です。
- **B地点：** 実況が30℃以上の日は【2日(30), 3日(33), 4日(32), 5日(30), 6日(31)】の5日間。
    - このうち予報が30℃未満（見逃し）だったのは、**2日（予報28）と6日（予報29）の2回**です。
- **結論：** どちらも見逃した回数はBのほうが多いです。「B地点の方が低い」というのは **誤（×）** です。

## 4. まとめ

すべての判定を並べると以下のようになります。

- (a) **正**（どちらも引き算の合計がプラスになる）
- (b) **誤**（B地点の方が誤差が小さくて優秀）
- (c) **誤**（見逃した回数はBのほうが多い）

これに綺麗に一致するのは、選択肢 **②** ですね！

> 
> - **平均誤差（ME）のワナ：** 大外ししても、プラスとマイナスが同じくらいあれば「ゼロ」になります。だからMEは**「予報の性格（高めが好きか、低めが好きか）」**を見るものと覚えましょう。
> - **ガチの優秀さ（RMSE）：** プラスマイナスを帳消しにさせないために2乗する、**「言い訳を許さない本当の実力テスト」**です。数字のブレが少ないBの方が当然優秀（RMSEが小さい）になります！

# 問15：正解1

- **気圧面の高度**：地上からその気圧（300hPaや100hPa）がある場所までの「実際の高さ（メートル）」のことです。
- **寒帯前線ジェット気流**：高緯度側に位置する、上空の強い風の帯のことです。
- **北極振動（AO）**：北緯60°を挟んだ北極域と中緯度（日本など）の間で、気圧がシーソーのように「一方が高ければもう一方が低い」という逆の関係を持つ現象です。
    - **プラスの位相**：北極域の気圧が低く、中緯度の気圧が高い状態です。
    - **負の位相**：北極域の気圧が高く、中緯度の気圧が低い状態です。

## 2. 「ルール」をシンプルに提示する（変数の整理）

この問題を解くためのルールは、以下の通りに整理されます。

- **300hPa面の高度は、おおよそ8000m台から9000m台**（高度約10km付近）になります。
- **100hPa面の高度は、おおよそ15000m台から16000m台** になります。
- **北極振動が負の位相のときは、北極域と中緯度との気圧差が平年よりも小さくなるため、寒帯前線ジェット気流が弱い傾向**になります。
- **【最重要】上空で等高度線の波（リッジ）が発達して高度が平年より高くなると、空気の柱が厚くなり、その下にある地上のシベリア高気圧が強化**されます。これが、上空の高度の平年偏差（プラス・赤色）と、地上の海面気圧の平年偏差（プラス・赤色）がほぼ一致する理由です。
